Cynthia Chen是产品设计师，从未写过代码。她用Replit和Claude，花两个月业余时间做出狗狗品种识别App"Dog-e-dex"，成功上架App Store。（Business Insider 2025.5）

前UX设计师Rebecca Beach用vibe coding制作数字产品（日记本、课程、工作簿），原本需要几周的工作现在2小时完成。她在Etsy和Shopify月销售额超过$20,000。（Business Insider 2025.8）

Otto's Grotto贴纸店店主Therese Waechter，用ChatGPT辅助文案和自动化流程，2024年收入比前一年翻倍。（Business Insider 2025.4）

## 人人都能成为 Builder

"Builder"这个词很难翻译，它比"程序员"的含义更广。

一个 Builder 是**能把想法变成现实的人**。

以前，成为 Builder 需要：
- 会写代码，或者
- 有钱雇人写代码

现在，成为 Builder 只需要：
- 有想法
- 能把想法说清楚
- 愿意花时间和AI一起迭代

::: tip 核心观点
**Vibe Coding 不是让你成为程序员，而是让你成为 Builder。**

你的价值不在于"会写代码"，而在于"知道要做什么"和"能判断做得好不好"。
:::


## 你可能还有顾虑

### "我怕做不好"

别担心。Vibe Coding 的核心精神就是**"试错成本很低"**。

做得不好？改呗。
改崩了？让AI重来。
一个方案不行？换个思路再说。

你会在后面的章节学到，怎么保存进度、怎么回退版本，让你放心大胆地试。

### "我的想法会不会太简单"

恰恰相反。**简单的想法更容易成功**。

很多人失败是因为想法太复杂，而不是太简单。

后面的"心法"章节会详细讲这个——如何设计一个"刚刚好"的第一版。

### "AI做的东西能用吗"

能用。

对于个人工具、原型验证、小团队内部使用，AI生成的代码完全够用。

当然，如果你要做一个面向百万用户的产品，那就需要更专业的方案——但那是后话，也是进阶版教程会讲的内容。


## 也要知道边界

**适合用Vibe Coding的场景**：
- 个人工具、原型验证、内部使用
- 快速试错、验证想法
- 学习和探索

**需要更专业方案的场景**：
- 面向大量用户的生产系统
- 涉及敏感数据、金融交易的应用
- 需要长期维护和迭代的大型项目

::: warning 安全提醒
根据Wiz Research 2025年报告：
- 1/5使用vibe coding的组织存在系统性安全风险
- 常见问题：硬编码密码、客户端存储敏感信息、缺少访问控制
- 约36%的AI生成代码可能包含安全漏洞

如果你的应用涉及用户数据、支付等敏感场景，需要额外的安全审查。
:::

Vibe Coding是一个强大的起点，但如果你想走得更远，后面还有很多东西要学。
